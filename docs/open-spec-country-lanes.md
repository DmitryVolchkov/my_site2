## 1. Флажки без названий (только даты)

**Решение:** базовый режим — флажок с датой, без заголовка.

Обязательные дополнения:

- Бейдж количества «×N», если на дате в полосе больше одного события (пример: 01.09.1939 — 5 записей). Плотность дня — значимый сигнал, её нельзя скрывать.
- Hover-подсказка с заголовком события (`headline`) — без клика.
- Полная карточка — по клику (как сейчас в TimelineJS).

Открытое: LOD — показывать ли заголовки вместо дат при сильном зуме (мало событий в viewport).

## 2. Пустая дата при явном запросе (поиск по дате / клик по оси)

**Решение:** комбинированное поведение.

- Пунктирная вертикаль остаётся на запрошенной дате как «якорь» запроса.
- Подсказка с ближайшими событиями в обе стороны, **по каждой полосе отдельно**.
- Расстояние до ближайшего — главный элемент подсказки («через 11 месяцев»), а не мелкая подпись: снап без явного расстояния создаёт ложную смежность событий.
- Переход к ближайшему — только по клику, не автоматически.
- Если в радиусе N дней пусто — предлагать расширение: «в марте 1942 — 4 события» (день → месяц → год), с явной пометкой, что это события месяца/года, а не дня.

Данные: до cutover на Canvas — из уже загруженного `timeline_data.json`; после — `GET /api/timeline/nearest` (требования к нему по точности, интервалам и полосам зафиксированы в canvas-спеке).

## 3. Точность дат при поиске

**Решение:** событие с точностью месяц/год или `approximate` «покрывает» все дни своего периода.

- Поиск 15.03.1940 сразу находит событие «март 1940» — с пометкой «возможно относится» — без ручного расширения.
- Правило для `approximate`: событие попадает в выдачу окрестности даты (радиус — уточнить, ориентир ± точность даты).
- Иначе поиск систематически теряет все приблизительно датированные события предвоенного периода.

## 4. Пустая полоса при просмотре

**Решение:** пустота не должна читаться как «ничего не происходило».

- Никаких призрачных меток на самой шкале (не путать с реальными данными).
- Минимум: счётчик в подписи полосы — «Германия · 2 события».
- Желательно: индикатор покрытия периода архивом (где наполнение велось, а где нет).
- Формулировка для UI: «в архиве», а не «в истории».

## 5. Место ≠ участник (модель данных) — блокирующий вопрос

**Проблема:** `country_name` — это *где* произошло событие. При полосах-странах:

- 22.06.1941 (нападение Германии на СССР) лежит как «СССР» → не появится в полосе Германии.
- Пакт Молотова–Риббентропа виден только одной стороне.

Полосы сравнения стран по месту события систематически искажают картину.

**Решение:** ввести семантику «участники события» — связь M:N (по образцу `event_groups`, например `event_countries` с ролью: место / участник). Полосы стран строить по участникам, фильтр географии — по месту. Решить до Canvas-рендера: миграция дешевле сейчас.

Связанное: составные значения («Литва, Латвия, Эстония») нормализовать при миграции; до неё — правило split по запятой.

## 6. Предпосылки по контенту (до тестирования UI)

- Внести таксономию групп (см. ниже) и разнести 17 событий — иначе фильтр групп не по чему проверять.
- Расщепить составные `country_name`.
- Учитывать при наполнении: полоса страны с 2 событиями почти пуста — приоритет конвейера внесения дат по слабо покрытым странам сравнения.

### Таксономия групп (принята)

База — 17 верхнеуровневых рубрик IPTC Media Topics, свёрнутых в 5 основных групп с подгруппами:

| Основная группа | Подгруппы |
| --- | --- |
| **Общество** | культура и искусство; спорт; религия; стиль жизни и досуг; человеческий интерес; происшествия и катастрофы |
| **Политика** | право и преступность |
| **Экономика** | труд и занятость; экология и окружающая среда |
| **Военные действия** | архивная специфика (не из IPTC): фронтовые операции; оборона; оккупация; потери; партизанское движение |
| **Наука** | образование; здоровье и медицина; погода (метеорология) |

Спорные отнесения (зафиксированы осознанно): здоровье и медицина — в науку, не в общество; экология — в экономику (ресурсы), не в науку.

**Схема:** реализовано миграцией `013_add_group_parent.sql` — `groups.parent_id` + 23 записи (5 основных, 18 подгрупп; grp-0002…grp-0023). Применяется при старте сервера.

**Правила иерархии (следствия для UI и данных):**

- Событию назначается **наиболее специфичная** группа (подгруппа, если подходит; основная — если подгруппы нет).
- Фильтр групп на главной — двухуровневый: выбор основной группы включает все её подгруппы.
- Агрегаты «×N» (лестница, п. 8) и счётчики полос считают события подгрупп в счёте родителя.
- Полоса по основной группе (canvas, семейство «Группы») агрегирует события своих подгрупп; полоса по подгруппе — только свои.
- Форма события и справочник в админке — двухуровневый список с отступами (блок A порядка работ).

**Судьба группы «История» (`grp-0001`):** это заглушка «всё подряд», а не тема — в таксономию не входит. После разнесения 17 событий по новым группам (ориентир: 5 — военные действия, 12 — политика; спорные ev-0018, ev-0024) группу вывести из употребления через API (не прямым SQL — аудит и пересборка JSON).

## 7. Поиск по дате: навигация и коллизии

**Решение:** поиск — это навигация, а не только якорь.

- Ввод даты панорамирует шкалу: дата — в центре viewport, с автозумом до уровня «месяц» (видны дни).
- Дата **с событиями**: выделяется сам флажок (поверх соседей, акцентная обводка, соседи приглушены). Пунктир не показывается.
- Дата **без событий**: пунктирный якорь + ближайшие по полосам (п. 2). Пунктир — индикатор именно «пусто».
- Выделенный/найденный флажок всегда выигрывает коллизию — рисуется поверх и полностью.
- Для плотных участков без поиска — минимальная разводка из спеки MVP (второй ряд внутри полосы / микросдвиг).

## 8. Лестница уровней зума (LOD)

**Решение:** дискретные уровни детализации; ось — сама навигация.

| Уровень | Видимый блок | Деления оси | Флажок = | Клик по флажку/делению |
| --- | --- | --- | --- | --- |
| Век | 100 лет | половины века | агрегат полувека «×N» | → половина века |
| Половина века | 50 лет | все годы | агрегат года «×N» | → год |
| Год | 12 месяцев | все месяцы | агрегат месяца «×N» | → месяц |
| Месяц | все дни | дни | реальные события (дата + ×N) | → карточка |

Правила:

- Выше уровня «месяц» флажки — **всегда агрегаты** по единице оси. Коллизии соседних дат исчезают по построению (максимум 12 флажков на полосу на уровне «год», 50 — на «половине века»).
- Агрегат = счётчик + кнопка спуска. Счётчик считается по полосе с учётом активного фильтра групп.
- Пустой агрегат не рисуется — пустота остаётся информативной (см. п. 4, полоска покрытия).
- Колесо/pinch — плавные; переключение детализации по порогу: ширина единицы уровня < ширины флажка → схлопывание на уровень выше.
- Подъём: breadcrumb над шкалой («1900–1949 › 1939 › сентябрь»), клик по крошке или подписи оси.
- Поиск даты (п. 7) — прыжок сразу на уровень «месяц».

**Точность дат на лестнице:** событие с точностью «месяц» при спуске в дни некуда ставить — приколачивать к 1-му числу нельзя (ложная датировка). Решение: кликабельная плашка в шапке полосы «+N с точностью до месяца». Аналогично для точности «год» на уровне месяцев и `approximate`.

## 9. Точечные и интервальные флаги

**Решение:** форма флага определяется данными события автоматически.

- **Только начальная дата** (`start_*`) → точечный флажок: одна нога на дате.
- **Заполнена конечная дата** (`end_*`) → флаг с двумя ногами: левая нога на дате начала, правая — на дате окончания, перекладина показывает протяжённость. Ярлык — диапазон дат («30.11.39 – 13.03.40»).
- Ноги всегда стоят на фактических датах на любом уровне зума. Если граница интервала выходит за видимый блок — нога рисуется пунктиром со стрелкой («продолжается за краем»).
- Короткий интервал, сжимающийся до пары пикселей на верхних уровнях, — минимальная ширина фигуры, детали в hover-подсказке.

Правила взаимодействия с лестницей зума (п. 8):

- Агрегаты «×N» считают **только точечные** события. Интервал не растворяется в агрегате месяца/года — рисуется отдельной фигурой поверх уровня (иначе война «схлопнется» во флажок месяца начала).
- Интервал рисуется на каждом уровне, где его диапазон пересекает viewport.

Правила взаимодействия с поиском по дате (пп. 2, 7):

- Интервальное событие **покрывает** все даты между ногами: поиск даты внутри диапазона отвечает «длится: <событие> (начало – конец)», а не «пусто».
- Ответ «на дату пусто» и ближайшие события — только если нет ни точечных попаданий, ни покрывающих интервалов в полосе.
- Это закрывает открытый вопрос № 3 спеки Canvas («полоска interval vs два маркера»): принята полоска-интервал с двумя ногами.

Точность дат: для интервала точность начала и окончания независимы (`start_date_precision` / `end_date_precision`); нога с точностью «месяц»/«год» на уровне дней показывается как зона неопределённости, не приколачивается к первому числу (правило п. 3 действует на каждую ногу отдельно).

Контент-предпосылка: в тестовой базе (17 событий) поля `end_*` не заполнены ни у одной записи — для проверки UI заполнить окончания у реальных длительных событий (зимняя война, германо-польская война) через админку с источниками.

## 10. Составные события в полосах и агрегатах

**Решение:** агрегаты и счётчики считают только основные события.

- Вторичные события (`attached_to` заполнен) **не входят** в «×N» агрегатов и счётчики полос отдельно от основного — иначе плотность завышается. Они видны внутри объединённой карточки основного события, как сейчас.
- Полоса (участники/группы) определяется атрибутами **основного** события; вторичные собственных маркеров не порождают.
- `related_events` («см. также») на шкале не визуализируются на этапах 1–2 — осознанно отложено. Кандидат на будущее: линии-связи между полосами (реакция одной страны на действие другой) — записать как открытый вопрос при проектировании этапа 3.

## 11. Мобильный (тач) и легенда обозначений

**Проблема:** решения пп. 1, 8, 9 опираются на hover, которого на таче нет; условных обозначений уже пять (одна нога, две ноги, пунктирная нога, «×N», пунктирный якорь).

**Решение:**

- Тач-эквивалент hover — двухшаговый тап: первый тап по флажку — подсказка (заголовок/диапазон/«кликните, чтобы раскрыть»), второй — действие (спуск по лестнице или карточка). Альтернатива для проверки на прототипе: панель-подсказка, закреплённая под шкалой.
- Легенда: кнопка «?» в menubar открывает список обозначений; при первом посещении — краткая подсказка (флаг в `localStorage`).
- Дополняет раздел «Тач и мобильные жесты» canvas-спеки.

## 12. Состояние таймлайна в URL (deep-linking)

**Решение:** всё состояние просмотра сериализуется в URL и восстанавливается при открытии.

- В query/hash: уровень лестницы, период, полосы (страны), фильтр групп, выделенная дата. Пример: `?lanes=ussr,ger&group=war&level=month&at=1939-09-17`.
- Кнопка «назад» браузера двигается по истории навигации лестницы (History API) — подъём/спуск не «выбрасывает» с сайта.
- Ссылкой можно поделиться («смотри: СССР и Германия, сентябрь 1939»); закладки работают.
- Закладывать в этап 2 вместе с полосами — ретрофит дороже.
